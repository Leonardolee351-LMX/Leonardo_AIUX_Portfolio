# 4天作品集冲刺协同清单（互联网大厂投递版）

> 目标岗位：体验设计师 / 交互设计师 / 传统产品经理  
> 冲刺目标：4天内产出可投递的作品集网站 + 对齐简历叙事版本（可讲、可看、可追问）

## 0. 总体成功标准（Day 4 验收）

- [ ] 网站可在线访问，PC与手机可读
- [ ] 至少 3 个核心项目有完整叙事链路（背景-问题-过程-决策-结果-反思）
- [ ] 简历中的项目描述与作品集页面表达一致
- [ ] 每个项目都能在 3 分钟内讲清楚
- [ ] 每个项目至少有 1 个可被面试官追问的关键决策点

---

## 1. 网站最小结构（先完成再精修）

- [ ] `首页 Home`：个人定位、能力标签、项目入口、联系方式
- [ ] `项目总览 Work`：项目卡片 + 标签筛选（可简化为静态分类）
- [ ] `项目详情 Project`：统一叙事模板（建议先做 3 个）
- [ ] `简历页 Resume`：摘要版经历 + 下载入口 + 外链（若已有 `resume.html` 则直接优化）

建议优先保证信息可读与叙事完整，视觉动效控制在最低必要范围。

## 4. 项目优先级建议（先打穿，再扩展）

- [ ] P0：最能体现“问题定义 + 设计决策 + 业务结果”的项目
- [ ] P1：能体现“跨团队协作 / 从0到1 / 技术理解”的项目
- [ ] P2：展示多样性的补充项目（时间不足时可降级为简版）

## 5. 风险与应对（冲刺期）

- [ ] 风险：内容写不完  
      应对：先保证 3 个主打项目完整，其他项目做卡片摘要
- [ ] 风险：视觉细节耗时过高  
      应对：冻结视觉系统，优先内容和信息层级
- [ ] 风险：没有量化数据  
      应对：补“过程指标/代理指标/用户反馈”作为证据
- [ ] 风险：临近投递焦虑导致频繁推翻  
      应对：每日只允许一次结构性改版，其他只做增量优化
