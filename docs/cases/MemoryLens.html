MemoryLens

Apple Vision Pro 智能寻物助手 · AI Native 交互设计与实现

Role

AI Native 产品设计师 / 前端开发

Time

2025.07 · 黑客松快速开发

Platform

visionOS + Python WebSocket + Coze

Context

MemoryLens 的切入场景是老年人或高频忙碌用户的“日常物品遗忘”。项目目标是把“看见物体”升级成“理解用户正在找什么”，并给出实时、可执行的定位反馈。

问题定义：不是单纯目标检测，而是“视觉识别 + 语义对话 + 空间反馈”的连续体验。

System Design

  • 前端载体：Vision Pro 端使用 SwiftUI + RealityKit 构建混合现实交互。
  • 识别模块：Core ML 执行实时物体检测。
  • 通信链路：WebSocket 负责图像与状态实时传输。
  • AI服务：Coze 工作流处理语义理解与对话反馈。
关键决策：优先把“语音问询 -> 空间定位 -> 音频反馈”闭环打通，再逐步扩充识别类别。

Core Iterations

第一版只输出识别框，用户知道“识别到了”，但不知道“与我有什么关系”；第二版加入语音问答后，找物路径变清晰；第三版优化音频提示与视觉高亮同步，提升了在 MR 场景中的可理解性。

从工程上看，项目在有限时间内实现跨端协同（VisionOS + Python + AI 服务），证明了架构可扩展。

Result

48h+黑客松高压开发节奏

Reflection

这个项目强化了我的一个方法论：AI 交互不该只停在聊天界面，而要结合具体情境形成“动作可执行”的反馈链路。未来如果继续迭代，我会优先补充用户测试与多物体冲突场景策略。
