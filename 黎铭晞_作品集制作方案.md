# 黎铭晞 · 作品集网站制作方案

> **目标**：在周六日前上线一个体面的个人作品集网站  
> **可用时间**：每天 3-4 小时，总计约 10-12 小时  
> **核心策略**：先做骨架再填血肉，优先保证"体面上线"而非"完美主义"

## 二、信息架构（网站结构）

### 推荐结构：单页滚动 + 项目详情页

```
首页（单页长滚动）
├── Hero区（我是谁 + 一句话定位）
├── 核心能力标签
├── 项目速览（4张卡片，点击进详情）
├── 关于我（教育背景 + 联系方式）
└── Footer

项目详情页 × 4（独立页面）
├── 项目概览（时间/角色/工具/链接）
├── 背景与问题
├── 设计过程（过程稿 + 决策）
├── 最终产出
└── 反思与收获
```

### 为什么不用复杂结构？
- 时间不够做多页面动画
- KIMI的面试官更看重**内容质量**而非**动画炫技**
- 单页滚动在手机上体验更好

## 五、技术选型建议

### 推荐方案：React + Tailwind + shadcn/ui（1小时搭骨架）

**为什么？**
- 你已经会用TRAE/Lovable，React上手快
- 组件化结构，适合"先搭骨架再填内容"
- 部署简单（Vercel一键部署）

**替代方案（如果时间更紧）**
- **Framer**：可视化编辑，2小时出稿，但自由度低
- **Notion + Super.so**：最快（30分钟），但不够体面
- **Carrd**：单页网站，1小时搞定，适合极简风格

### 推荐的技术栈
```
框架：React + Vite
样式：Tailwind CSS
组件：shadcn/ui
动画：Framer Motion（简单淡入即可）
部署：Vercel
图片：存放在public文件夹或使用图床
```

## 九、给你的一些建议

1. **不要追求完美**：面试官看的是你的思维和过程，不是网站的动画有多炫
2. **过程稿是加分项**：手绘草图、Miro截图、Figma迭代版本，比最终成品更有说服力
3. **讲好决策故事**：每个项目至少有一个"当时我有两个选择，我选了A因为..."的段落
4. **量化一切**：能用数字就不用形容词（40+页、500+问卷、80%认可、5天48小时）
5. **链接打通**：作品集网站里的项目，都要能跳到你的Figma/GitHub/线上Demo
